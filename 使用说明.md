# 个人网站使用说明（beautiful-jekyll 版）

这是给你自己看的操作手册,不用懂技术,照着做就行。

---

## 一、网站由哪些文件组成

你日常**只会碰下面这几个**,其它一堆文件(`_layouts`、`_includes`、`assets` 等)是模板的"发动机",**永远别动**。

| 文件 | 作用 |
|------|------|
| `_config.yml` | ⭐ 全站设置:网站名、导航、邮箱、**配色风格** |
| `index.html` | ⭐ 首页内容:你的介绍、作品、联系方式 |
| `blog.html` | 博客列表页(一般不用改) |
| `_posts/` 文件夹 | ⭐ 你写的每一篇博客文章放这里 |

> ⚠️ 重要:这套模板**不能再双击 `index.html` 本地预览了**。它必须由 GitHub 在线生成。
> 流程变成:**改文件 → 推送到 GitHub → 等 1~2 分钟 → 刷新网址看效果**。

---

## 二、改首页内容(介绍 / 作品 / 联系方式)

打开 `index.html`,最上面 `---` 之间是标题,下面就是正文。
正文用 **Markdown** 写,规则就三条好记:

- `## 文字` = 中标题(比如「关于我」)
- `### 文字` = 小标题(比如某个作品名)
- `- 文字` = 列表的一项
- `**文字**` = 加粗

照着现有的样子改文字即可。

---

## 三、写一篇新博客

1. 进 `_posts` 文件夹,**复制**里面已有的那篇 `.md` 文件。
2. 把文件名改成「日期-名字」格式,例如 `2026-07-01-my-post.md`(**日期格式必须是 年-月-日**)。
3. 打开它,改最上面的 `title`(标题)、`subtitle`(副标题),然后在下面写正文。
4. 保存 → 推送 → 等一两分钟,博客页就会出现这篇文章。

---

## 四、换风格 / 换颜色

打开 `_config.yml`,找到「配色」那一段,每行后面都有中文注释告诉你管哪里,改颜色值即可(颜色值就是 `#` 开头那串,可以让 AI 助手帮你配)。
**改风格不会影响你的文字内容**,放心改。

---

## 五、怎么让修改显示到网上

改完文件保存后,网上不会自动更新,要"推送"一次:

- 用 **GitHub Desktop**:打开它 → 左边列出你改了哪些文件 → 下方填一句话(如"更新作品")→ 点 **Commit to main** → 再点右上角 **Push origin**。
- 等 1~2 分钟,刷新网址 `https://sunyufeilaw.github.io` 就能看到。

---

## 六、改坏了怎么办

- 如果网站打不开或样子乱了,多半是 `_config.yml` 里**缩进/冒号被弄乱**了,或文章文件名日期格式不对。
- 用 GitHub Desktop 右键文件选 **Discard changes(放弃修改)**,可退回上次正常状态。
- 实在搞不定,把情况告诉 AI 助手帮你修。
