# 个人网站使用说明（beautiful-jekyll 版）

这是给你自己看的操作手册,不用懂技术,照着做就行。

---

## 一、网站由哪些文件组成

你日常**只会碰下面这几个**,其它一堆文件(`_layouts`、`_includes`、`assets` 等)是模板的"发动机",**永远别动**。

| 文件 | 作用 |
|------|------|
| `_config.yml` | ⭐ 全站设置:网站名、**导航菜单**、邮箱、**配色风格** |
| `index.html` | ⭐ 首页内容:你的介绍、作品、联系方式 |
| `_posts/` 文件夹 | ⭐ 你写的每一篇文章放这里 |

> ⚠️ 重要:这套模板**不能再双击本地预览了**,必须由 GitHub 在线生成。
> 流程是:**改文件 → 推送到 GitHub → 等 1~2 分钟 → 刷新网址看效果**。

---

## 二、改首页内容(介绍 / 作品 / 联系方式)

打开 `index.html`,最上面 `---` 之间是标题,下面就是正文。
正文用 **Markdown** 写,规则就几条好记:

- `## 文字` = 中标题(比如「关于我」)
- `### 文字` = 小标题(比如某个作品名)
- `- 文字` = 列表的一项
- `**文字**` = 加粗

照着现有的样子改文字即可。

---

## 三、写一篇新文章,并归到某个分类

1. 进 `_posts` 文件夹,**复制**里面已有的那篇 `.md` 文件。
2. 把文件名改成「日期-名字」格式,例如 `2026-07-01-my-post.md`(**日期必须是 年-月-日**)。
3. 打开它,改最上面的 `title`(标题)、`subtitle`(副标题)。
4. ⭐ **关键:用 `tags` 给文章归类。** 你想让它出现在哪个菜单,就写那个菜单的名字:

   | 想归到哪个菜单 | 就这样写 |
   |----------------|----------|
   | 法律内容 → 民法 | `tags: [民法]` |
   | 法律内容 → 刑法 | `tags: [刑法]` |
   | 法律内容 → 案例 | `tags: [案例]` |
   | 非法律内容 → 生活随笔 | `tags: [生活随笔]` |

   > 一篇文章也可以同时属于多个分类,用逗号隔开:`tags: [民法, 案例]`。
5. 在下面正文区域写内容,保存 → 推送 → 等一两分钟,文章就会出现在对应菜单里(也会出现在「全部文章」里)。

---

## 四、⭐ 怎么新增一个分类菜单(超简单,不用建文件)

所有分类都集中在一个「文章分类」总览页里,所以**加分类不再需要新建任何文件**。
比如你想在「非法律内容」下加一个「读书笔记」,只需一步:

打开 `_config.yml`,找到 `非法律内容:` 那一段,在下面加一行(注意对齐前面那个 `-`):

```
  非法律内容:
    - 生活随笔: "/tags/#生活随笔"
    - 读书笔记: "/tags/#读书笔记"     # 新增这一行,# 号后面填新分类名
```

保存、推送。完工——菜单里就多了「读书笔记」,以后给文章贴上 `tags: [读书笔记]`,它就会自动归到这个分类里。

> - 想在「法律内容」下加(比如「行政法」),做法完全一样,只是加在 `法律内容:` 那一段下面。
> - 删分类也一样:把对应那一行删掉即可,**不会留下多余文件**。

---

## 五、换风格 / 换颜色

打开 `_config.yml`,找到「配色」那一段,每行后面都有中文注释告诉你管哪里,改颜色值即可(颜色值就是 `#` 开头那串,可以让 AI 助手帮你配)。
**改风格不会影响你的文字内容**,放心改。

---

## 六、怎么让修改显示到网上

改完文件保存后,网上不会自动更新,要"推送"一次:

- 用 **GitHub Desktop**:打开它 → 左边列出你改了哪些文件 → 下方填一句话(如"新增分类")→ 点 **Commit to main** → 再点右上角 **Push origin**。
- 等 1~2 分钟,刷新网址 `https://sunyufeilaw.github.io` 就能看到。

---

## 七、改坏了怎么办

- 如果网站打不开或样子乱了,多半是 `_config.yml` 里**缩进/冒号被弄乱**了,或文章文件名日期格式不对。
- 用 GitHub Desktop 右键文件选 **Discard changes(放弃修改)**,可退回上次正常状态。
- 实在搞不定,把情况告诉 AI 助手帮你修。
